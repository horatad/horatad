# โปรเจกต์สกัด rules โหราศาสตร์ไทย → kb_tals format

## เป้าหมาย
แปลงตำราโหราศาสตร์ไทย → JSON rules → merge เข้า BIBLE knowledge base

---

## ลำดับการทำงาน (ทำตามนี้ทุก session)

1. **อ่าน** `astro_progress.md` → รู้ว่าค้างที่ไหน และ id ถัดไปคืออะไร
2. **อ่าน** `astro_few_shot.md` → จำ format ทั้ง 7 type
3. **อ่าน** `astro_schema.json` → เช็ค domain enum ก่อนใส่ค่า
4. **สกัด rules** จากข้อความที่ user ส่งมา
5. **output** เป็น JSON array พร้อม merge

---

## Output format (สำคัญ)

ส่ง output เป็น JSON array เสมอ:

```json
[
  { "id": "R291", ... },
  { "id": "R292", ... }
]
```

**วิธี merge เข้าระบบ:** user รัน `node workers/kb_merge.mjs <file.json>` หรือ paste JSON ตรงๆ

---

## ข้อตกลงสำคัญ

| กฎ | รายละเอียด |
|---|---|
| **ไม่แปลความเพิ่ม** | เก็บ source_quote เสมอถ้ามี exact quote จากตำรา |
| **ไม่แน่ใจ source_type** | ใช้ `"INFERRED"` เสมอ |
| **domain** | ต้องมาจาก enum ใน astro_schema.json เท่านั้น ห้ามแต่งใหม่ |
| **ข้อความ 1 ตอน** | อาจแตกเป็นหลาย rule ได้ตามเนื้อหา |
| **ไม่ต้องถามกลับ** | เริ่มสกัดได้ทันทีเมื่อได้รับข้อความ |
| **duplicate check** | ถ้า meaning คล้าย rule ที่มีอยู่แล้ว → เพิ่ม notes ว่า "possible duplicate of RXXX" |
