- **helpers อยู่ใน engine.js (ไม่แยก relations.js)** — เก็บ master dictionary ไว้ที่เดียวกับ KASET_MAP/EXALT_MAP เพื่อหา reference ได้ง่าย และ engine.js comment เดิมห้ามแก้ "logic คำนวณดาว" — helper ใหม่ไม่ใช่ raw computation
- **ไม่แก้ kb.json ในรอบนี้** — schema V2.4 ต้อง user/expert approve ก่อน ถ้า migrate ก่อน lock-in ผิดจะรื้อยาก (CLAUDE.md: "Architecture lock-in ระยะยาว → ถามก่อน")
- **CASE_STUDY ติดเป็น match → noise** — แยก rule_use field แทน rule_type เพราะ rule_type เดิมใช้เป็น semantic tag (DEFINITION/HOUSE_CONCEPT) ไม่ใช่ pipeline filter
- **`global_conditions` แทน planet-level "evil_count"** — รวมในตำแหน่งเดียวต่อ rule ไม่ใช่ duplicate ต่อ planet (95 rules มี 3+ required → จะกระจาย global_conditions ทุก condition เป็น noise)
- **`predictions[]` array แทน split `p` ด้วย regex** — user/expert ระบุ semantic boundary ตอน migration ไม่ใช่ heuristic split (ที่อาจตัดผิดความหมาย "นุ่มนวล, เสน่ห์" → ควรรวมเป็นกลุ่ม)
