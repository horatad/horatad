## WHY LOG
- **Source extraction แทน expert migration** — kb.json V2.3 ที่ Gemini extract มา lossy มาก (rules หาย, conditions ขาด, CASE_STUDY ปนกัน). 100 source files อยู่ใน repo แต่ยังไม่ถูกใช้. Claude API อ่าน docx ได้โดยตรง → extraction ถูกกว่าและครบกว่าการให้ expert นั่งแก้ทีละ rule

- **ไม่ทับ kb.json เดิม** — output ไปที่ kb_v24.json แยก เพื่อให้ app ยังทำงานได้ระหว่าง review extraction quality
- **ปิด 3 rules false positive (3a)** — "rules หายได้เพราะ 100 documents ไม่ช่วย" — user confirm ว่า rules ที่ไม่มีประโยชน์ควรออก ดีกว่ามีแล้ว noise
