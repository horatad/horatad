## WHY LOG
- **เพิ่ม accuracy A/B/C/D/F แทน confidence number** — เกรดแบบโรงเรียนสื่อสารง่ายกับ user (อ่านเข้าใจทันที) ขณะ confidence 0.85 ต้องตีความ
- **birth time = optional** — Astrotheme match rate 1.24% หมายความว่าถ้าบังคับ time_utc → ใช้ data ได้แค่ 588/47508 records (1%) ในทางตรงข้าม accuracy=D 46K records ใช้ได้ (เพราะ validate หน้างาน)
- **TARGET 50K → 100K** — distinct jd ยังไม่ saturate (20K/47K = 56% duplicate) แสดงว่า query series มี jd ซ้ำเยอะ → ขยาย target จะได้ jd ใหม่อีก
- **lat/lng = 0 ไม่ block** — engine คำนวณ planet position ใช้ jd + time_utc พอ ส่วน lagna ต้องการ lat/lng แต่ recordings ไม่บังคับมี (UI ให้ user กรอกตอน load)
- **ไม่ลบ records accuracy=F (10 ตัว)** — เก็บไว้เพื่อ trace bug "07:16" placeholder ในอนาคต ถ้าเจอเพิ่มขึ้นมาก = scraper regress
