## 1. โครงสร้างภาษาโหราศาสตร์ไทย (Thai Astro Syntax)

### 1.1 สูตรกฎพื้นฐาน
```
[ดาว] + [สภาวะ/ตำแหน่ง] → [ผล/นิสัย]
```
ตัวอย่าง:
- `อาทิตย์(1) กุมลัคนา → หยิ่ง รักหน้า ไม่ยอมเสียหน้า`
- `เสาร์(7) อยู่ราศีมังกร(9) → เกษตร (มั่นคง ให้ผลช้าแต่ยั่งยืน)`
- `เสาร์(7)+ราหู(8) สัมพันธ์กัน → โชคใหญ่`

### 1.2 สูตรกฎสลับ (Quality Reversal)
```
[คุณภาพ] + [คุณภาพ] + เล็งกัน → [คุณภาพตรงข้าม]
```
ตัวอย่าง:
- `เกษตร + เกษตร + เล็ง → ประเกษตร (ดีก่อน แล้วเสื่อม)`
- `นิจ + นิจ + เล็ง → อุจ (ไม่ดีก่อน แล้วดี)`

### 1.3 สูตร Transit
```
เมื่อ[ดาวจร] + [ตำแหน่งเดินทาง] + กระทบ[ดาวพื้น] → [ผลชั่วคราว]
```
Markers: เมื่อ / ขณะที่ / ตอนที่ / ในปีที่ / เมื่อดาว / โคจร / ดาวเดิน

## 8. การแยกกฎออกจาก Text (Extraction Logic)

### 8.1 กฎ NATAL_ATOMIC (tier 1)
- Subject: ดาวเดียว (planet_ids=[1]) 
- Predicate: in_sign / กุมลัคนา / สัมพันธ์ลัคนา / เป็นตนุเศษ / เป็นตนุลัคน์
- Outcome: คุณสมบัติ/นิสัย/โชคชะตา

### 8.2 กฎ NATAL_COMBINATION (tier 2)
- Subject: ดาว 2+ ดวง (planet_ids=[X,Y])
- ทดสอบ: เอาดาวหนึ่งออก กฎยังมีความหมายไหม? ถ้าไม่ → combination
- Outcome: emergent meaning ที่ไม่เคยมีในดาวเดียว

### 8.3 กฎ TRANSIT_NATAL (timing)
- มี transit marker
- ดาวจรกระทบดาวพื้น
- ผลชั่วคราว

### 8.4 ตัดสิน rule_use
```
"match"       = กฎที่ใช้พยากรณ์ได้ทันที
"principle"   = หลักการที่ abstract จาก N กรณี  
"case_study"  = ตัวอย่างเฉพาะบุคคล
"reference"   = นิยาม/อ้างอิง ไม่ใช่กฎพยากรณ์
```

## 10. Ground Truth Tests

### CH036 — Transit คู่มิตร = โชคใหญ่
**Principle**: เมื่อดาวจร N คู่มิตรเปิดใช้งานพร้อมกัน → โชคใหญ่  
**กรณีบวก**: กรณีที่ 1-5 พิสูจน์ว่า transit คู่มิตรหลายคู่พร้อมกัน = โชคใหญ่ (ขนาดต่างกันตาม N คู่)  
**กรณีลบ**: เสาร์+อังคาร (คู่ศัตรู) transit → ทุกขลาภ ไม่ใช่โชค

**PASS**: กฎที่ extract ได้ต้องสามารถ score CH036 ว่าเป็น positive outcome ได้  
**FAIL**: ถ้ากฎระบุเสาร์+อังคาร → โชค

## 13. WHY LOG (เหตุผลการตัดสินใจสำคัญ)

### 13.1 ทำไมต้อง BIBLE_memory.md
Claude เรียนรู้ภาษาโหราศาสตร์ไทยได้เร็วแต่ลืมเมื่อ session ใหม่  
→ ไฟล์นี้ = long-term memory เฉพาะ domain vocabulary + extraction rules  
ต่างจาก handoff (= task state) เพราะไฟล์นี้คือ **ความรู้ที่ไม่ expire**

### 13.2 ทำไมไม่ใช้ master_dict.js เป็น memory
master_dict.js = machine-readable สำหรับ LLM extraction system  
BIBLE_memory.md = human+Claude-readable explanation พร้อม context และ why

### 13.3 ทำไม token ≠ word ในภาษาไทยโหราศาสตร์
LLM tokenize subword units เช่น "บาปเคราะห์" อาจแตกเป็น "บาป" + "เคราะห์"  
→ แต่ความหมายต้องอ่านเป็นคำประสม ไม่ใช่คำแยก  
→ master_dict.js ช่วยบอก LLM ว่าคำประสมใดห้ามตัด

### 13.4 ทำไมถึงมี Triple Extraction (Groq/Typhoon/Claude)
