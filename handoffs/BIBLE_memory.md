## 1. โครงสร้างภาษาโหราศาสตร์ไทย (Thai Astro Syntax)

### 1.1 สูตรกฎพื้นฐาน
```
[ดาว] + [สภาวะ/ตำแหน่ง] → [ผล/นิสัย]
```
ตัวอย่าง:
- `อาทิตย์(1) กุมลัคนา → หยิ่ง รักหน้า ไม่ยอมเสียหน้า`
- `เสาร์(7) อยู่ราศีมังกร(9) → เกษตร (มั่นคง ให้ผลช้าแต่ยั่งยืน)`
- `เสาร์(7)+ราหู(8) สัมพันธ์กัน → โชคใหญ่`

### 1.2 สูตรกฎสลับ (Quality Reversal)
```
[คุณภาพ] + [คุณภาพ] + เล็งกัน → [คุณภาพตรงข้าม]
```
ตัวอย่าง:
- `เกษตร + เกษตร + เล็ง → ประเกษตร (ดีก่อน แล้วเสื่อม)`
- `นิจ + นิจ + เล็ง → อุจ (ไม่ดีก่อน แล้วดี)`

### 1.3 สูตร Transit
```
เมื่อ[ดาวจร] + [ตำแหน่งเดินทาง] + กระทบ[ดาวพื้น] → [ผลชั่วคราว]
```
Markers: เมื่อ / ขณะที่ / ตอนที่ / ในปีที่ / เมื่อดาว / โคจร / ดาวเดิน

## 6b. ⚠️ Number Disambiguation — กฎสำคัญมาก

ตัวเลข **อารบิก (1-10)** และ **ไทย (๑-๑๐)** ในตำราโหราศาสตร์ไทย = planet_id ได้  
**แต่ต้องดูบริบทก่อนเสมอ** — เลขเดียวกันในประโยคเดียวอาจหมายถึงคนละอย่าง

```
ตัวอย่าง: "คนชื่อ 1 สอบได้ที่ 1 เพราะมี 1 ได้ตำแหน่งอุจในดวงชะตา"
- "คนชื่อ 1"   → ชื่อคน     ❌ ไม่ใช่ planet_id
- "สอบได้ที่ 1" → ranking    ❌ ไม่ใช่ planet_id  
- "มี 1 ได้อุจ" → อาทิตย์(1) ✅ planet_id
```

**สัญญาณว่าเป็น planet_id:**
- "ดาว N" / "ดวง N" — ตามหลังคำว่าดาว/ดวง
- "(N)" — ในวงเล็บตามหลังชื่อดาว เช่น "อาทิตย์ (๑)"
- "N กุมลัคนา" / "N อยู่ราศี" / "N สัมพันธ์กัน" — ทำหน้าที่เป็น subject ของกริยาดาว
- "N เป็นเกษตร/อุจ/นิจ" — อยู่ในบริบทคุณภาพดาว

**ไม่ใช่ planet_id ถ้า:**
- "บทที่ N" / "ข้อ N.N" / "ตอนที่ N" — chapter/section
- "ที่ N" / "อันดับ N" — ranking
- "N วัน/ปี/เดือน" — time unit
- "ราศี N" / "ภพ N" / "เรือน N" — เลข index ราศี/ภพ
- "N %" / "N กรณี" / "N คู่" — ตัวเลขทั่วไป

**Safe rule**: ถ้าไม่มีสัญญาณ planet_id ชัดเจน → ไม่ extract เป็น rule

## 8. การแยกกฎออกจาก Text (Extraction Logic)

### 8.1 กฎ NATAL_ATOMIC (tier 1)
- Subject: ดาวเดียว (planet_ids=[1]) 
- Predicate: in_sign / กุมลัคนา / สัมพันธ์ลัคนา / เป็นตนุเศษ / เป็นตนุลัคน์
- Outcome: คุณสมบัติ/นิสัย/โชคชะตา

### 8.2 กฎ NATAL_COMBINATION (tier 2)
- Subject: ดาว 2+ ดวง (planet_ids=[X,Y])
- ทดสอบ: เอาดาวหนึ่งออก กฎยังมีความหมายไหม? ถ้าไม่ → combination
- Outcome: emergent meaning ที่ไม่เคยมีในดาวเดียว

### 8.3 กฎ TRANSIT_NATAL (timing)
- มี transit marker
- ดาวจรกระทบดาวพื้น
- ผลชั่วคราว

### 8.4 ตัดสิน rule_use
```
"match"       = กฎที่ใช้พยากรณ์ได้ทันที
"principle"   = หลักการที่ abstract จาก N กรณี  
"case_study"  = ตัวอย่างเฉพาะบุคคล
"reference"   = นิยาม/อ้างอิง ไม่ใช่กฎพยากรณ์
```

## 10. Ground Truth Tests

### CH036 — Transit คู่มิตร = โชคใหญ่
**Principle**: เมื่อดาวจร N คู่มิตรเปิดใช้งานพร้อมกัน → โชคใหญ่  
**กรณีบวก**: กรณีที่ 1-5 พิสูจน์ว่า transit คู่มิตรหลายคู่พร้อมกัน = โชคใหญ่ (ขนาดต่างกันตาม N คู่)  
**กรณีลบ**: เสาร์+อังคาร (คู่ศัตรู) transit → ทุกขลาภ ไม่ใช่โชค

**PASS**: กฎที่ extract ได้ต้องสามารถ score CH036 ว่าเป็น positive outcome ได้  
**FAIL**: ถ้ากฎระบุเสาร์+อังคาร → โชค

### 13.5 Pattern สำคัญที่พบใน Batch 2 (ch015-ch029)

**ตนุเศษ vs กุมลัคนา vs ตนุลัคน์ — สามระดับ**
- ตนุเศษ = พื้นนิสัยโดยรวม (ไม่ขึ้นคุณภาพ ไม่ขึ้นภพ — ทุกที่ก็ยังเป็นนิสัยนั้น)
- กุมลัคนา = ลักษณะภายนอก (100% ชัด แต่หารด้วยจำนวนดาวกุม)
- ตนุลัคน์ = ตัวตนภายใน (100% = ดาวกุมทุกดวงรวมกัน ไม่หาร)
→ สามระดับนี้อาจเสริมหรือขัดกันได้ในคนเดียว

**การตรวจสอบลัคนา (Lagna Verification)**
- ถ้าพยากรณ์ไม่แม่น → ย้ายลัคนา±1 ราศี (ไม่สร้าง exception)
- ใช้ตรวจ: อาทิตย์ (ถือตัวไหม?), พุธ (ช่างพูดไหม?), อังคาร (ใจร้อนไหม?), เสาร์ (คิดมากไหม?)
- พฤหัสสัมพันธ์/ไม่สัมพันธ์ลัคนา = signal แรงมาก (แม่นยำสูง)
