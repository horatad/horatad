## WHY LOG
- **data/julian_all.json ใน repo เป็น primary storage** — GitHub raw เป็น CORS-free ไม่มีค่าใช้จ่าย ไม่ต้องการ token และ app อ่านได้โดยตรง CF D1 เหมาะกับ private/write-heavy data ที่จะทำในอนาคต
- **ลบ CF steps ออกจาก workflow** — CF token เป็น external dependency ที่ทำให้ทุก step หลังจากนั้นล้มเหลวเงียบๆ เมื่อ token ไม่ถูกต้อง แก้ด้วยการตัด dependency ออกเลยดีกว่า workaround
- **ENV VAR แทน process.argv ใน node -e** — `node -e "code" "$file"` ส่ง arg ไปที่ argv[2] ไม่ใช่ argv[1] bug นี้ทำให้ข้อมูล 436 records หายไปโดยไม่มี error ใดๆ
- **git pull --rebase ก่อน push** — workflow รันนาน 15+ นาที ระหว่างนั้น session เราอาจ push commit ใหม่ขึ้น main ทำให้ runner push reject โดย rebase แทน merge เพื่อ keep history สะอาด
- **Astrotheme→wiki (P3447) แทน wiki→Astrotheme** — ค้นหาคนที่มี Astrotheme ID โดยตรงได้ exact URL path ทันที ไม่ต้อง guess จากชื่อที่มีโอกาส miss ~70%
- **Reset progress.json** — เพราะ bug argv ทำให้ข้อมูลที่ scrape ได้ไม่เคยถูกบันทึกลง repo แม้ seen_qids จะมี 436 entries แล้ว ต้อง re-scrape ใหม่ทั้งหมด
- **ย้าย Astrotheme หลัง Commit progress** — ให้ enrich records จาก batch นี้ทันทีในรอบเดียวกัน ไม่ต้องรอ run ถัดไป
