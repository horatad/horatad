## WHY LOG (session นี้)

- **tz=7 default ไม่ใช่ null** — ถ้าใช้ `null` เป็น default จะต้องเช็ค null ทุก call site → เปลือง code + error-prone. `tz=7` default ทำให้ TS/Thai record ทุก call ที่ไม่ส่ง tz ยังถูกต้อง (backward-compatible โดยไม่ต้องแก้ call site เก่า)
- **_customTz state parallel กับ _customLng** — pattern เดียวกับ _customLng: set ใน _pickMemory/carousel/quickLoad, clear ใน reset/clearForm. ไม่มี UI input ให้ user set tz โดยตรง (เพียงพอตอนนี้ — tz ดึงจาก record ที่ import)
- **เก็บ tz=null ใน record ไม่ใช่ 7** — null = "ไม่รู้" (ใช้ default 7 ตอน calc). 7 = "รู้ว่าเป็น UTC+7 โดย user ระบุ". การแยกทำให้ future-proof ถ้าเพิ่ม UI ให้ user ระบุ tz ทีหลัง
