## WHY LOG (v1 — Phase 0 outputs)

- **เลือกทำ Phase 0 ทั้ง 7 task ใน session เดียว** — เพราะทุก task เป็น read-only audit (grep + Read) + ไม่ขึ้นต่อกัน → batch ได้, ประหยัด session overhead, ออก Phase 1 plan ครบรอบเดียว. Trade-off: handoff ยาว (acceptable เพราะมี TL;DR ใน KEY FINDINGS)

- **F1 (QR XSS) มอง low-medium ไม่ใช่ low** — ปกติ single-user XSS ไม่นับ vector แต่ QR import เป็น cross-trust boundary (link ที่ส่งมา from outside) → ใครก็ตามที่ส่ง URL แบบ unsigned H1 มาให้ user เปิด = inject ได้. Mitigation: 10-line _escHtml fix ใน P1-D + CSP backstop ใน P1-B

- **T-06 horatad-auth gate = cosmetic, ต้องชี้ให้ HORATAD owner เห็น** — ไม่ใช่ฟ้อง bug แต่ชัดเจนว่าระบบที่มีอยู่อาจ misalign กับ intent. ถ้า PIN เพื่อ gate cost (likely) → ต้อง refactor horatad-ai Worker ให้ require session. ถ้า PIN เพื่อ UX friction → document ให้ชัด ไม่ให้คนคิดว่ามันคือ security control

- **CSP Report-Only ก่อน enforce 1 สัปดาห์** — inline `onclick` ทั่ว script.js จะ violation ถ้า enforce ตรงๆ → break UI. Report-only มอง footprint ก่อน → Phase 2 ค่อยเลือก refactor (option E) vs `'unsafe-hashes'` (option D). ห้าม jump ไป enforce mode โดยไม่ผ่าน report-only

- **Wikidata compliance (T-04) ตรวจแล้ว COMPLIANT — ไม่ต้องแก้** — ต่างจาก expectation ของ v2 handoff ที่ note ว่า "ไม่มี audit ว่าเคารพ ToS หรือไม่". จริงๆ `julian_scraper.mjs` มีครบทุกอย่าง (User-Agent contact, exponential backoff, 1500ms gap, Retry-After respect). ปิด R-16 ได้เลย

- **gha audit พบ R-12 ปิดแล้ว** — workflow trigger design (no `pull_request_target`) ทำให้ fork PR ไม่มีทาง access secrets. ไม่ต้องเพิ่ม control. Free win.

- **script.js 392KB ไม่ใช่ emergency** — gzip ~95KB acceptable for SPA. parse time ~200ms mid-tier mobile = OK. Phase 2 code-split เก็บไว้สำหรับ when LCP measure ปรากฏปัญหาจริง — อย่าทำ over-optimization ก่อนวัด

- **Maskable icon = quick win แต่ effort ขึ้นกับมี image tool ไหม** — sandbox อาจสร้าง PNG ด้วย canvas ผ่าน Node script ได้ ถ้าทำ P1-C ใน session ถัดไป
