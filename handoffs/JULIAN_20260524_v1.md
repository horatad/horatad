## WHY LOG

- **Skip Phase 0 (migrate)** — user decision: julian_all.json ปัจจุบันไม่มี source provenance ที่แม่นแล้ว (COALESCE history หาย) → migrate จะได้ raw bucket แบบ approximate ไม่คุ้ม → เริ่ม raw fresh จากรอบ scrape ใหม่ดีกว่า, source-pure ตั้งแต่ต้น
- **Dual-write แทน replace** — Phase 1 production-safe (workflow เดิมยังอ่าน path เดิมได้) → migration gradual ไม่ต้อง coordinate กับ GUARD ทันที
- **julian_merge.mjs standalone (ไม่ wire workflow ตอนนี้)** — JULIAN scope ไม่แตะ .github/workflows/ (GUARD-owned) → ทำ standalone ที่ทดสอบได้ → ส่ง handoff request GUARD wire เข้า workflow ภายหลัง
- **_meta fields strip ใน merge output** — julian_all.json ส่ง HORATAD consume → ต้อง schema เดิม (ไม่มี _scraped_at/_bucket) → strip ใน applyBase()
- **Priority: seed > wiki_th > astrotheme > wikidata** — manual seed มี user judgment (สูจิบัตร/ครอบครัว); enrichment overlays preserve Wikidata base; last-write-wins สำหรับ Wikidata เพราะ same QID = same person
- **Latent bug ใน julian_empirical.mjs** — `execSync from 'fs'` เป็น invalid import (fs ไม่ export execSync) → file จะ crash on module load → ยังไม่ deployed (ไม่มี workflow trigger) → fix แล้ว ระวังถ้าจะใช้
