## WHY LOG

- **M5 เสร็จ V3.3.21** — _updateNavHeader ทำไปพร้อมกัน เพราะ depend กัน: compareMode===2 ถ้า header ยังอ่าน synastry จะแสดงผิด
- **M4 เสร็จ V3.3.20** — natal2 (63 จุด) → synastry ทั้งหมด รวมถึง line 2759 (event setter) ชั่วคราว, M5 จะแยก eventChart จาก synastry จริง
- **M4 แยก session** — natal2 ใช้ 3 roles ปนกัน (สมพงศ์/event/transit) ถ้า replace ผิดที่จะ break rendering ทั้งหมด → session แยกเพื่อ focus + grep verify ทุก occurrence
- **Bridge _viewMode/_outerState** — M2 ไม่ break rendering เดิมโดยยังคง map _compareMode→_viewMode ไว้ก่อน → M4/M5 จะทำให้ _viewMode obsolete
- **_transitCursor ใช้ _nowStr() format** — ปัจจุบัน field `y` (ไม่ใช่ `y_be`) เมื่อ init จาก _nowStr() เพราะ _nowStr() return `{d,m,y,t}` ตาม _era → M6 ต้อง normalize ให้เป็น y_be เสมอ
- **transit-unit popup position** — `bottom:80px` เพื่อไม่บัง transit-strip บนมือถือ → ปรับถ้า layout เปลี่ยน
- **_tsCalc ยังไม่ sync กับ _transitCursor** — _tsCalc() อ่าน DOM inputs โดยตรง ไม่ผ่าน _transitCursor → user กรอกวันในช่องแล้วกดคำนวณจรยังทำงาน แต่ _transitCursor ไม่ update → M3 minor fix ที่ยังค้างอยู่
