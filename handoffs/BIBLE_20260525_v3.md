## WHY LOG

- **POC scoring penalize REFERENCE rather than filter out**
  Could have filtered out REFERENCE entirely, but they have signal value (R248 = "ดาวกุมลัคนา → ลักษณะ" is REFERENCE but very useful). Negative score lets them appear if nothing else fits, but pushes them below specific-position rules.

- **Planet-presence bonus cap at +6 (not +20 from 10 planets)**
  Rule mentioning 10 planets shouldn't ipso facto beat a rule about 1 specific planet in 1 specific house. Cap forces scoring to depend on SPECIFICITY (pair matches, lagna-relation) not just BREADTH.

- **planet_positions เกตุ + มฤตยู: null vs empty arrays**
  `null` for uchcha/nicha = "concept does not apply to this planet" (semantic absence)
  `[]` for kaset/prakaset/mahachak = "concept exists but planet has none"
  These are different — null encodes "don't ask this question", [] encodes "the answer is none"

- **mahasatru: explicit empty + note (not omitted)**
  Could have omitted mahasatru entirely. Explicit empty + note = future contributor sees "we considered this, ตำราสุริยยาตร์ standard ไม่มี" → won't reopen the question.

- **house_rulers_by_lagna shipped pre-computed (not derived at runtime)**
  144 entries × ~30 bytes = ~4KB. Trivial cost. Benefits: human-verifiable, O(1) lookup, documents schema relationship explicitly. Trade-off: if signs[].ruler changes, must regenerate matrix — but signs[].ruler should be stable.

- **กุมภ์ uses ราหู as primary in matrix (not เสาร์)**
  signs[11].ruler_planet_id = "8" (ราหู), secondary = "7" (เสาร์). Matrix uses primary only to avoid 2-ruler-per-house ambiguity. Consumer needing เสาร์ interpretation must check signs[11].ruler_secondary_planet_id explicitly. Documented in `_skeleton_note`.

- **Schema migration deferred not executed**
  Migration touches ID scheme across 4+ entries × 2 schemes — high blast radius. Bridge fields (canonical_*_id) added this session make current usage safe. Real migration cheap to defer until 3rd consumer makes it necessary. Trigger conditions explicit in LOG so future sessions know when to act.
