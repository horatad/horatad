## ภาพรวม JULIAN (อ่านทุก session)

JULIAN คือ Search Engine + Database — มี 2 ตาราง:

```
Master Key Table                    Internet Table
────────────────────────────        ──────────────────────────────────
jd        INTEGER PRIMARY KEY       id        INTEGER PK (autoincrement)
planets   TEXT (JSON array[10])     jd        INTEGER (FK → Master Key)
          degrees 0-360, precision  name      TEXT
          เก็บล่วงหน้าทุกวัน         event_label TEXT
                                    type      TEXT (human|event)
                                    lat       REAL
                                    lng       REAL
                                    time_utc  TEXT (HH:MM หรือ null)
                                    relate_id TEXT (JSON array of JD)
                                    source    TEXT
                                    confidence REAL (0.0-1.0)
                                    notes     TEXT
```

**หลักการ:**
- Master Key = 1 row ต่อ 1 วัน (JD เดียวกันได้) — เก็บดาวล่วงหน้า ไม่ต้องคำนวณซ้ำ
- Internet Table = หลาย record ต่อ 1 JD ได้ (คนหลายคนเกิดวันเดียว, หลายเหตุการณ์)
- relate_id = array of JD ที่สัมพันธ์กัน เช่น [วันเกิด, วันตาย, วันเหตุการณ์สำคัญ]

**JULIAN → BIBLE:**
- BIBLE ส่ง JD มา → JULIAN คืน planets[10] (Master Key) + person/event data (Internet Table)
- BIBLE เลือกใช้เองว่าต้องการข้อมูลส่วนไหน (empirical_p, secondary rules, ฯลฯ)

## ⚠️ Decision Point — JD format (ต้องตัดสินใจก่อน build)

`script.js:_calcJD()` ใช้ modified formula ที่ไม่ใช่ standard Julian Day
- Standard JD: Jan 1 2000 CE = **2451545.0** (ใช้ทั่วโลก, interoperable กับ Wikipedia/databases อื่น)
- Internal JD: formula ใน script.js ให้ค่าต่างออกไป (ใช้ภายใน HORATAD ปัจจุบัน)

**แนะนำ:** JULIAN ควรใช้ standard JD — เพราะต้องดึงข้อมูลจาก internet และเชื่อมกับ source ภายนอก
แล้วแปลงกลับเป็น internal format เมื่อส่งให้ HORATAD/BIBLE

**รอ user ยืนยัน** ก่อน build Master Key generator

## Context สำคัญ

**planet position format:** script.js เก็บใน arc-minutes (0–21600 = 360°)
สำหรับ JULIAN ใช้ degrees (0–360) ง่ายกว่า — แปลง: `degrees = arcmin / 60`

**_calcJD() location:** script.js:60 — ถ้า copy ต้อง test ว่าให้ค่าถูกต้อง
standard JD formula ทางเลือก: `JD = 367Y - INT(7*(Y+INT((M+9)/12))/4) + INT(275*M/9) + D + 1721013.5`

**เป้าหมายระยะยาว:** JULIAN ส่ง planets[] + context ให้ BIBLE → BIBLE ใช้คำนวณ empirical_p ต่อ rule
(กี่คนที่มี config นี้ แล้วชีวิตเป็นแบบที่ rule บอกจริงๆ)
