## WHY LOG

- **Apply tool decision semantics — merge=true, separate=false production-ready**
  Decision rationale: `separate` produces N UNIQUE rules from a previously dupe group — each new rule still has only 1 source → not triangulated → can't claim production-ready. `merge` means user verified all wordings belong to same rule → triangulation by user authority = production-ready.

- **Separate fallback to per-wording split (when no source files loaded)**
  Without source files, can't reconstruct original condition_text per rule_id. Best-effort: split per-wording, emit warning, mark `decision_applied: "separate (fallback: per-wording)"` so audit trail is clear. User can re-run with source files for better results.

- **POC scoring: planet+house pair = +5 vs single planet = +2**
  Specific position (planet IN house) is much higher signal than "planet exists in chart". A REFERENCE rule mentioning all 10 planets shouldn't outrank an exact match like "อาทิตย์ในภพปุตตะ". Current weighting still favors REFERENCE because 10 × 2 = 20 > single pair match 5 — TODO penalize rule_use=reference in v2.

- **Fill signs structural from SIGNS.md is NOT hallucination**
  The original skeleton note said "Claude ไม่เติม content เพื่อป้องกัน hallucination" — but SIGNS.md (BIBLE memory) is verifiable ground-truth from ch001+ch007. Filling structural fields (ruler/element/nature) from a documented source isn't hallucination; it's pulling forward established knowledge. Narrative fields (represents/keywords) genuinely need user input — kept empty.

- **ID scheme mismatch flagged not migrated**
  Master_dict uses 1-12 (1=เมษ), KB canonical uses 0-11 (0=เมษ). Migrating now = breaking change across engine.js + every reader. Added `canonical_sign_id` bridge field instead — cheap, non-breaking, makes both schemes accessible. Real migration deferred to next major bump.

- **Pipeline closed but real triangulation still blocked on user**
  Step 1-5 (extract → merge → review → apply) all shipped. But `kb_merged.json` currently has 0 AUTO_VERIFIED rules because only 1 source loaded (kb_v24-3). Real triangulation = AUTO_VERIFIED count > 0 = requires user runs Groq + Typhoon extractions (browser tool, sandbox can't). This is the single blocker keeping the architecture from proving itself.
