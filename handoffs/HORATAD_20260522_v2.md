### State Variables ใหม่

```js
// Charts (renamed)
let natal = null;         // ดวงใน — was natal1
let synastry = null;      // ดวงสมพงศ์ที่เลือก
let eventChart = null;    // ดวงเหตุการณ์จรที่เลือก
let transit = null;       // ดวงจรที่เลือก (pos คำนวณจาก _transitCursor)

// Cursors — จำไว้ ไม่ reset เมื่อสลับ mode
let _synastryIdx = 0;           // index ใน _dbPersons()
let _eventIdx = 0;              // index ใน _dbEvents(natal.uid)
let _transitCursor = _today();  // {d,m,y_be,t} เริ่ม = วันนี้

// Display mode
let _compareMode = 0;    // 0=natal only | 1=synastry | 2=event | 3=transit
let _outerDisplay = 1;   // 0=hide | 1=data | 2=ราศี | 3=ภพ

// Transit unit
let _transitUnit = {type:'fixed', value:'day'};
// type='fixed'  → value: 'day'|'month'|'1y'|'5y'|'10y'|'50y'|'100y'
// type='planet' → value: 'moon'|'sun'|'mercury'|'venus'|'mars'
//                        |'jupiter'|'saturn'|'ketu'|'rahu'|'mrityu'
```

### Migration Table (เดิม → ใหม่)
```
natal1           → natal
natal2 (person)  → synastry
natal2 (event)   → eventChart
natal2 (transit) → transit
_viewMode (0/1)  → _compareMode (0-3)
_outerState(0-4) → data part: _compareMode | visual part: _outerDisplay
_reportTransitShow → derive: _compareMode===3 && _outerDisplay!==0
cycleMemory(dir)  → cycleOuter(dir) — aware of compareMode+activeTank
toggleView()      → setCompareMode(n)
toggleOuter()     → setOuterDisplay(n)
```

### Toggle1 (_compareMode) — 4 states

```
0 = natal only   → outer ring = null    → buildReport(natal)
1 = vs synastry  → outer ring = synastry.pos  → buildCompareReport(natal, synastry,  'สมพงศ์')
2 = vs event     → outer ring = eventChart.pos → buildCompareReport(natal, eventChart,'เหตุการณ์')
3 = vs transit   → outer ring = transit.pos   → buildCompareReport(natal, transit,   'ดวงจร')
```

**Report logic:**
- compareMode=0 OR outerDisplay=0 → buildReport(natal) เท่านั้น
- compareMode=1/2/3 AND outerDisplay≠0 → buildCompareReport(natal, outerChart, label)
- buildCompareReport: template เดียวสำหรับทุก compare type (ต่างกันแค่ header label)

---

### Toggle2 (_outerDisplay) — 4 states (visual only)
```
0 = ไม่แสดง outer ring  → report = natal only (ไม่เทียบ)
1 = แสดงตาม compareMode → outer ring = pos ของ synastry/event/transit
2 = ชื่อราศีจักร         → override visual เท่านั้น (รายงานยังตาม compareMode)
3 = ชื่อภพเรือน          → override visual เท่านั้น
```
⚠️ outerDisplay=0 → report = natal only (เหมือน compareMode=0) ตาม user confirm

### ลูกศร ◀ ▶ ตาม compareMode

| compareMode | action | cursor ที่จำ |
|---|---|---|
| 0 | วน natal ตาม `_activeTank` (private/julian/_julianFiltered) | _memScrollKey |
| 1 | `_synastryIdx ±1` → reload synastry | _synastryIdx (จำไว้) |
| 2 | `_eventIdx ±1` → reload eventChart | _eventIdx (จำไว้) |
| 3 | `_transitCursor` ±unit → recalc transit | _transitCursor (จำไว้) |

**compareMode=0 + activeTank:**
- _activeTank='private' → วน private tank (เหมือน cycleMemory เดิม)
- _activeTank='julian' → วน _julianFiltered (รายการที่ filter ไว้)

**State persistence:** _synastryIdx, _eventIdx, _transitCursor ทั้ง 3 ต้องบันทึกใน localStorage
→ เมื่อสลับ mode แล้วสลับกลับ ต้องเจอ state เดิม

### Transit Unit Popup

UI (single-select, 1 ตัวเลือกเท่านั้น):
```
[ระยะเวลา]
● วัน  ○ เดือน  ○ 1ปี  ○ 5ปี  ○ 10ปี  ○ 50ปี  ○ 100ปี

[ย้ายราศี]
○ จันทร์  ○ อาทิตย์  ○ พุธ    ○ ศุกร์   ○ อังคาร
○ พฤหัส  ○ เสาร์    ○ เกตุ   ○ ราหู   ○ มฤตยู
```

**Algorithm "ย้ายราศี":**
```js
// step size per planet (for iteration)
const PLANET_STEP_HOURS = {
  moon:1, sun:24, mercury:24, venus:24, mars:24,
  jupiter:168, saturn:336, ketu:336, rahu:336, mrityu:720
};

function _transitNextSignChange(dir, planet) {
  const planetIdx = {moon:1, sun:0, mercury:2, venus:3, mars:4,
                     jupiter:5, saturn:6, ketu:7, rahu:8, mrityu:9};
  const stepH = PLANET_STEP_HOURS[planet];
  let cursor = {..._transitCursor};
  const startSign = _getPlanetSign(cursor, planetIdx[planet]);
  let safetyLimit = 0;
  while(safetyLimit++ < 10000) {
    cursor = _addHours(cursor, dir * stepH);
    if(_jdFromCursor(cursor) < JD_CE0 || _jdFromCursor(cursor) > JD_CE2700) {
      _showToast('เกินขอบเขต CE 0–2700'); return;
    }
    if(_getPlanetSign(cursor, planetIdx[planet]) !== startSign) {
      _transitCursor = cursor;
      _recalcTransit();
      return;
    }
  }
}
```

Limit: JD ของ CE 0 = ~1721426 | CE 2700 = ~2816788
→ ชน limit: toast "เกินขอบเขต ค.ศ. 0" หรือ "เกินขอบเขต ค.ศ. 2700"
