## WHY LOG

- **Clean-branch pattern** — feature branch = staging ชั่วคราวเท่านั้น ไม่ใช่ long-running ดังนั้น reset หลัง ff ทำให้ครั้งต่อไปเป็น ff ตรงเสมอ ไม่ต้อง rebase และป้องกัน foreign commits ติดมา
- **Trigger-based memory แทน time-based** — "อัปเดตก่อนจบ session" ทำให้ forget ง่าย; trigger แต่ละ event (พบ quirk ใหม่, เปลี่ยน schema) บังคับ update ทันที ข้อมูลสดกว่า
- **Memory files แยกออกจาก handoffs** — handoff มี expiry (task state เปลี่ยนทุก session), memory ไม่มี expiry (domain knowledge สะสม) → แยกไฟล์ป้องกัน archive ทิ้ง
- **Soft enforcement แทน hard** — hard block บน git พัง context ทั้งหมด; soft (banner + refusal) ให้ Claude ใช้ judgment ได้ตาม context จริง (emergency fix ข้าม project ได้ถ้าจำเป็น)
- **File ownership table** — ก่อนมีตาราง Claude ต้อง infer เอง → miss ง่าย; ตารางชัดเจน ลด cognitive load และป้องกันแก้ผิดไฟล์
