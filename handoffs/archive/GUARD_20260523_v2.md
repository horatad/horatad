## WHY LOG (v2 additions)

- **เลือกทำ P1-A + P1-D + P1-G batch ก่อน P1-B** — Phase 1 quick wins ที่ effort ต่ำ + ไม่มี breaking risk → push main ครั้งเดียวก่อน เพื่อให้ user เห็น CI status (gitleaks scan ทำงาน) ก่อนแก้ headers ที่อาจมี side effect

- **CSP เริ่มที่ Report-Only โหมด + `'unsafe-inline'`** — script.js generates inline `onclick=` handlers หลายร้อยจุด ถ้า enforce strict CSP ทันที = break ทุก click. Pragmatic baseline (allow inline) ดีกว่าไม่มี CSP เลย — defense in depth ระดับ 1 ของ 3 ที่ตั้งใจ. Phase 2 เลือก strict refactor (option E) หรือ accept pragmatic ตามไฟล์ headers_draft

- **Extract inline `<script>` แม้ใช้ `'unsafe-inline'`** — ลด attack surface 1 ระดับ (Cross-Site Script Inclusion guard); ทำให้ source map cleaner; auth-pin.js แยก review/audit ได้ในไฟล์เดียว → benefit-to-effort สูงมาก (5 min refactor)

- **Maskable icon ใช้ background `#0d1117` ไม่ใช่ `#b8860b`** — manifest มี 2 color: background (#0d1117 = dark) + theme (#b8860b = gold). Per PWA spec, maskable background ต้องตรงกับ background_color เพราะ adaptive icon ตัดมุม → ขอบที่โผล่ต้องเข้ากับ splash screen สีเดียวกัน. ถ้าใช้สีทอง พื้นจะตัดมุมเป็นทองไม่กลืนกับ context

- **เลื่อน P1-E ออกไป** — Option A/B/C เป็น user decision ที่กระทบ UX (modal ก่อน share). GUARD แนะนำ C แต่ implement ต้อง user confirm. ไม่ใช่ [ทดลองใช้] (sandbox ทำได้) แต่ [BLOCKED] (รอ user OK) — ตามกฎ CLAUDE.md "ห้ามจับงาน BLOCKED มาทำเอง"

- **Generate maskable icon ผ่าน Pillow ใน sandbox** — sandbox ไม่ได้มี ImageMagick แต่ pip ติดตั้ง Pillow ได้. 80% safe-zone เป็น Android Adaptive Icon spec (icon ถูก crop เป็น circle/squircle/rounded-square ตาม launcher). source logo เดิมถ้าใช้ตรง = ถูกตัดขอบ; safe-zone padding ป้องกัน

- **Rotation reminder ใช้ Q1/Q2/Q3/Q4 dedupe** — workflow_dispatch สามารถ trigger หลายครั้งใน Q เดียว แต่ scheduled trigger เกิดครั้งเดียว/ไตรมาส. Dedupe ป้องกันคน manual fire แล้วได้ issue ซ้ำ. Labels = `guard, rotation` (project-scoped) ไม่ใช่ generic
