## WHY LOG

- **เปลี่ยน framing จาก strict 100CH → ธรรมเนียม broader (mid-session)** — user correct ผมที่ over-suspicious กับ Gemini output. ทำให้ architecture เปลี่ยนทั้งหมด จาก "audit each rule" → "triangulation auto-verify" → ลด user effort 80%

- **Element fingerprint vs wording text** — user proposal: rules ที่ elements ตรงกัน = same rule (wording ต่างได้). insight: triangulation ทำงานได้แม้ LLM rephrase ตำราคนละแบบ

- **KB equalizer principle** — กับ KB ใส่ context, Typhoon/Claude/Groq output ใกล้กัน → app ไม่ผูก vendor (cost flexibility + vendor independence)

- **Deep-parse ทำใน regex ไม่ต้องใช้ LLM** — vocabulary โหราศาสตร์ไทยจำกัด + standardized → regex parse lagna_relation/inline quality/house/sign แม่นยำ + เร็ว + ฟรี

- **Sync risk: browser tool ↔ Node script** — kb_extract.html embeds duplicates ของ kb_add_fingerprint.mjs + kb_deep_parse.mjs. แก้ที่หนึ่ง = แก้ที่สอง. Documented in code comment + LOG warning

- **Memory architecture pivot** — session นี้พิสูจน์ว่า memory แค่ "เก็บไว้" ไม่พอ ต้อง enforce อ่าน — แต่ enforcement ที่ทำได้ในตอนนี้คือ INDEX MUST READ FIRST + reminder ใน CLAUDE.md (hooks-based enforcement = future work)
