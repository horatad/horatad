## WHY LOG (decision log สำหรับ project นี้)

- **เลือกชื่อ CIA** — ตรงกับ security triad classic (Confidentiality/Integrity/Availability) + พอดีกับสิ่งที่ user ถาม (3 หัวข้อ: security, performance, trade-off). Acronym จำง่าย, ฟัง intelligence (เหมาะกับ project ที่ "วิเคราะห์ตรวจสอบ")

- **Phase 0 = Discovery only ก่อน change อะไร** — ถ้า fix ก่อนวัด baseline จะไม่รู้ว่าดีขึ้นจริงไหม. กฎ "Measure before / Measure after" จากปรัชญา CIA

- **ไม่ทำ login / OAuth ตอนนี้** — แอป Horatad ทำงานได้โดยไม่มี account, การเพิ่ม login เพิ่ม attack surface ฟรี (R-04 หาย แต่ R-13/R-15 มาแทน). ทำเมื่อมี business need จริง

- **CSP report-only ก่อน enforce** — Vanilla JS app มี inline style/event handler หลายจุดที่เราอาจไม่รู้. CSP ที่เข้มเกินไปอาจ break ทันที. soft-launch 1 สัปดาห์ดู console violations ก่อน enforce

- **15MB julian_all.json ไม่ลบทันที** — Phase 2 ตัดสินใจตอนวัด LCP จริง. ถ้า lazy-load อยู่แล้ว (ไม่อยู่ใน CORE_ASSETS / SW cache) → ไม่กระทบ initial load มาก → priority ลด

- **Rate limit ที่ 30 req/min ไม่ใช่ 10/min** — user จริงทดสอบหลายดวงพร้อมกัน (กรอก-ดูพยากรณ์-ลบ-กรอกใหม่) อาจเกิน 10 ใน 1 นาที. 30 generous พอ + กัน bot scrape

- **ไม่ตั้ง pen-test ตอนนี้** — งบ + scope แอป astrology ไม่คุ้มกับ pen-test pro ($2K-10K). เก็บไว้รอ scale
