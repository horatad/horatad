### T-01 ⭐ จาก HORATAD — QR URL privacy (R-07)
**Source:** `handoffs/HORATAD_20260522_v3.md:64` `[BLOCKED] QR URL privacy — รอ Option A/B`  
**Status:** BLOCKED (รอ user decision Option A vs B)  
**Risk ID:** R-07 (User-data scrape via QR URL)  
**Context:** QR share สร้าง URL ที่ encode วันเกิด+ชื่อ user ลง query string — ใครเปิด link ก็เห็น  
**Options ที่ค้าง:**
- Option A — **Plain encode** (status quo): ง่าย, share ได้ทุกที่, ไม่มี server, แต่ใครเห็น URL = เห็นข้อมูล
- Option B — **Encrypted** (URL + secret in fragment `#key`): privacy ดีกว่า, แต่ break ถ้า fragment หาย, ต้อง JS decrypt
- Option C ใหม่ที่ CIA เสนอเพิ่ม — **Friendly disclosure**: คงรูปแบบเดิม (Option A) แต่เพิ่ม UI ที่บอกชัด "ใครเปิด link จะเห็น ชื่อ+วันเกิด ของท่าน" + ปุ่ม "สำเนา link" ที่เตือนทุกครั้ง
**CIA recommendation:** Option C (friendly disclosure) เป็น quick win — privacy ที่ user controllable + ไม่ break compatibility กับ link เก่า. Option B ค่อยทำใน Phase 2 ถ้า user ต้องการ
**Connects to:** CIA-P1-D (Privacy Notice ใน index.html)
**Owner:** CIA (Phase 1)
